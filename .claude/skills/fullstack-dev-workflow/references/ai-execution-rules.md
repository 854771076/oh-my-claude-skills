# AI自动化执行规范与使用示例

---

## AI自动化执行增强规范

| 序号 | 规范名称 | 规范内容 |
|------|----------|----------|
| 1 | 指令锁定 | 明确「严格按照[文档路径]执行」 |
| 2 | 分阶段锁定 | 未通过准出校验禁止进入下一环节 |
| 3 | 计划先行强制 | 阶段3开发前必须先生成计划并存档，无计划不编码 |
| 4 | 用户交互强制 | 阶段-1必须使用 AskUserQuestion 进行技术栈确认 |
| 5 | 自检自动化 | 每环节完成后输出自检报告 |

---

## 异常终止条件

以下情况立即终止：

| 序号 | 终止条件 |
|------|----------|
| 1 | 新项目未完成技术栈调研直接进入开发 |
| 2 | 技术栈版本未锁定（使用"最新版"等模糊描述） |
| 3 | 用户未确认技术选型方案 |
| 4 | **开发代码前未生成实施计划** |
| 5 | **计划未存档至 `docs/plans/`** |
| 6 | **计划未通过用户确认** |

## 使用示例

### 新项目完整流程（阶段-1 → 阶段0 → 阶段1...）

```
"我要开发一个AI知识库问答系统，请帮我完成从技术选型到开发的完整流程"
```

### 阶段-1：技术栈调研（自动生成AGENTS.md）

```
"请为我调研适合开发电商后台管理系统的技术栈，我倾向使用TypeScript"
```

### 老项目直接初始化（分析现有代码，生成AGENTS.md）

```
"这是一个已有的React项目，请按阶段0搭建文档驱动开发体系"
```

### 阶段3：开发前生成计划

```
"请为用户模块开发生成实施计划，基于 docs/02-frontend-design/pages/user/index.md 设计文档"
```

### 老项目需求迭代

```
"请按阶段5处理需求变更：用户模块增加头像上传功能"
```

### 仅生成/更新AGENTS.md

```
"请分析当前项目并生成AGENTS.md文件"
```

### 查看历史计划

```
"请列出 docs/plans/archived/ 中的所有已完成计划"
```

## 开发流程速查

### 新功能开发流程

```
1. 需求文档化（阶段1）→ prd.md, srs.md
2. 架构设计（阶段2.1）→ system-architecture.md
3. 前端设计（阶段2.2）→ 页面设计文档
4. 后端设计（阶段2.3）→ 接口设计文档
5. 代码开发（阶段3）→ 基于设计文档编码
6. 测试验收（阶段4）→ 测试报告
```

### 需求变更流程

```
1. 变更申请 → 分配变更单号
2. 全量备份 → .version-history/
3. 影响分析 → 依赖矩阵查询
4. 文档更新 → 按优先级同步
5. 一致性校验 → 校验报告
6. 代码变更 → 基于新文档编码
```

## 自检报告模板

每环节完成后必须输出自检报告：

```markdown
## AI自检报告

### 校验项清单

| 校验项 | 校验结果 | 备注 |
|--------|----------|------|
| 文档完整性 | 通过/不通过 | [详情] |
| 索引闭环 | 通过/不通过 | [详情] |
| ID规范性 | 通过/不通过 | [详情] |
| 版本一致性 | 通过/不通过 | [详情] |
| 依赖矩阵同步 | 通过/不通过 | [详情] |

### 发现问题

| 问题ID | 问题描述 | 修复建议 |
|--------|----------|----------|
| [如有问题] | [描述] | [建议] |

### 校验结论

- 校验状态：通过/不通过
- 下一步操作：[允许进入下一环节/需修复后重新校验]
```
