## 四、设计规范与编码标准

### 4.1 代码风格规范

| 规范项 | 规范内容 |
|--------|----------|
| 格式化工具 | ESLint + Prettier / Black / gofmt |
| 配置文件路径 | 【配置文件路径】 |
| 自动校验 | pre-commit hook + CI检查 |

### 4.2 命名规范

| 类别 | 规范 |
|------|------|
| 文件命名 | kebab-case（如 user-service.ts） |
| 变量命名 | camelCase（如 userInfo） |
| 常量命名 | UPPER_SNAKE_CASE（如 MAX_RETRY_COUNT） |
| 类/组件命名 | PascalCase（如 UserService） |
| 接口命名 | I前缀或无前缀（如 IUserService） |
| 函数命名 | camelCase，动词开头（如 getUserById） |

### 4.3 目录规范

```
src/
├── frontend/           # 前端代码
│   ├── components/     # 公共组件
│   ├── pages/          # 页面组件
│   ├── hooks/          # 自定义hooks
│   ├── services/       # API服务
│   ├── stores/         # 状态管理
│   └── utils/          # 工具函数
├── backend/            # 后端代码
│   ├── controllers/    # 控制器层
│   ├── services/       # 服务层
│   ├── models/         # 数据模型
│   ├── repositories/   # 数据访问层
│   ├── middlewares/    # 中间件
│   └── utils/          # 工具函数
```

### 4.4 API规范

| 规范项 | 规范内容 |
|--------|----------|
| API版本 | 路径版本化 /api/v1/ |
| 响应格式 | JSON，统一结构 { code, message, data } |
| 错误码体系 | HTTP状态码 + 业务错误码 |
| 文档规范 | OpenAPI 3.0，自动生成 |
| 认证方式 | JWT Bearer Token |

### 4.5 安全规范

| 规范项 | 规范内容 |
|--------|----------|
| 传输安全 | HTTPS强制 |
| 认证机制 | JWT / OAuth2 |
| 密码存储 | bcrypt / argon2 加密 |
| 输入校验 | 类型校验 + 业务校验 |
| SQL注入防护 | ORM参数化查询 |
| XSS防护 | 输出编码 + CSP |
| CSRF防护 | Token校验 |
| 日志脱敏 | 敏感字段不记录 |
